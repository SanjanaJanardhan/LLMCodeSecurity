# LLM Code Security Research

**Authors:** Shriya Sudhakar, Sanjana Janardhan — Cornell CS 6158, Fall 2025

A research project studying how different prompting strategies affect the security of code generated by LLMs. We benchmarked Claude and GPT across multiple prompting approaches using the CWEval framework, which tests generated code against real Common Weakness Enumeration (CWE) vulnerabilities.

**Short version of findings:** Few-shot prompting improved security. Newer models were more capable but also surprisingly easier to manipulate into generating insecure code.

---

## Research Questions

- Does prompting strategy (direct, few-shot, feedback-driven) significantly affect the security of LLM-generated code?
- Do newer, more capable models generate more secure code by default?
- Can insecure code generation be induced through targeted prompting?

---

## Prompting Strategies Evaluated

| Strategy | Description |
|---|---|
| `direct` | Zero-shot — just ask the model to write code |
| `few_shot` | Provide examples of secure code before the prompt |
| `few_shot_insecure` | Provide examples of insecure code (adversarial) |
| `feedback_driven` | Feed pytest failure output back to the model iteratively |

---

## Models Tested

- `claude-sonnet-4-5` (Anthropic)
- `claude-haiku-3` (Anthropic)
- `gpt-4.1` / `gpt-5.1` (OpenAI)

## Getting Started

### Prerequisites
- Python 3.8 (for Claude prompting)
- Python 3.10 (for running CWEval tests)
- Anthropic API key and/or OpenAI API key

### Setup
```bash
conda create -n llm-security python=3.8
conda activate llm-security
pip install -r requirements.txt
```

### Running Claude experiments
Add your Anthropic API key to `cwe_eval_generate.py` line 25, then:
```bash
python scripts/cwe_eval_generate.py gen \
  --ppt="direct" \
  --n=10 \
  --temperature=0.1 \
  --eval_path="scripts/evals/my_run" \
  --langs='["py"]'
```

`--ppt` options: `direct`, `fewshot`, `feedback`

### Running GPT experiments
Switch to Python 3.10, add your OpenAI API key to `cwe_eval_generate_openai.py` line 25, then:
```bash
python scripts/cwe_eval_generate_openai.py gen \
  --ppt="fewshot" \
  --n=10 \
  --use_openai=True \
  --model=gpt-5.1 \
  --temperature=0.1 \
  --eval_path="scripts/evals/my_gpt_run" \
  --langs='["py"]'
```

### Running evaluation
```bash
# Run tests against generated code
python scripts/run_eval_clear_pycache.py

# Generate per-CWE breakdown
python scripts/analyze_per_cwe.py
```
Update `eval_path` in each script to match your run directory.
